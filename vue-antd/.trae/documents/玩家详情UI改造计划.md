# 玩家详情UI改造计划

## 1. 设计目标

将当前的修炼页面改造为玩家详情UI，包含多个tab页签，修炼功能作为其中一个tab。

## 2. 页面结构设计

### 2.1 整体布局

* **顶部区域**：玩家基本信息（头像、名称、等级、经验条）

* **中间区域**：Tab页签导航

* **内容区域**：根据选中的tab显示不同内容

* **底部区域**：通用操作按钮

### 2.2 Tab页签设计

* **属性**：显示玩家基本属性（攻击、防御、生命等）

* **修炼**：当前修炼页面的功能

* **背包**：玩家物品管理

* **技能**：玩家技能系统

* **任务**：玩家任务系统

## 3. 实现步骤

### 3.1 安装必要组件

确保项目中已安装Ant Design Vue的Tabs组件

### 3.2 修改修炼页面为玩家详情UI

1. 保留原有修炼功能代码
2. 添加Tabs组件
3. 创建多个tab内容区域
4. 将原有修炼代码迁移到修炼tab中

### 3.3 实现Tab内容

1. **属性tab**：显示玩家基本属性
2. **修炼tab**：原有修炼功能
3. **背包tab**：简单的背包UI框架
4. **技能tab**：简单的技能UI框架
5. **任务tab**：简单的任务UI框架

### 3.4 调整路由

确保玩家详情UI的路由配置正确

## 4. 技术实现

### 4.1 使用Ant Design Vue的Tabs组件

```vue
<a-tabs v-model:activeKey="activeTab">
  <a-tab-pane tab="属性" key="attributes"></a-tab-pane>
  <a-tab-pane tab="修炼" key="cultivation"></a-tab-pane>
  <a-tab-pane tab="背包" key="inventory"></a-tab-pane>
  <a-tab-pane tab="技能" key="skills"></a-tab-pane>
  <a-tab-pane tab="任务" key="tasks"></a-tab-pane>
</a-tabs>
```

### 4.2 迁移修炼功能

将原修炼页面的代码迁移到修炼tab中，保持原有功能不变

### 4.3 实现其他tab内容

为其他tab创建基本的UI框架，后续可以逐步完善功能

## 5. 预期效果

* 玩家进入玩家详情页面时，默认显示属性tab

* 可以通过tab切换查看不同内容

* 修炼功能保持原有逻辑不变

* 页面布局清晰，交互流畅

## 6. 后续扩展

* 完善背包、技能、任务等功能

* 添加更多tab页签（如社交、成就
