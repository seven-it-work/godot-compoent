# 调整游戏交互流程

## 目标
- 让首页默认显示地图页面（外出系统）
- 玩家需要在地图上选择地点后，点击"进入修炼"按钮才能进入修炼界面
- 修炼界面添加"返回地图"按钮，方便玩家切换

## 修改内容

### 1. 修改初始系统状态
- **文件**: `src/store/gameStore.ts`
- **修改**: 将`currentSystem`初始值从`"training"`改为`"outdoor"`

### 2. 修改外出系统组件
- **文件**: `src/components/OutdoorSystem.vue`
- **修改**: 
  - 将"返回修炼系统"按钮改为"进入修炼"
  - 点击事件改为进入修炼系统

### 3. 修改修炼系统组件
- **文件**: `src/components/TrainingSystem.vue`
- **修改**: 
  - 添加"返回地图"按钮
  - 点击事件返回外出系统

### 4. 调整导航栏逻辑
- **文件**: `src/App.vue`
- **修改**: 
  - 调整导航栏按钮的显示逻辑
  - 确保玩家只能通过按钮切换系统，而不是直接从导航栏切换

## 预期效果
1. 游戏初始化后直接进入地图页面
2. 玩家在地图上移动选择地点
3. 选择好地点后，点击"进入修炼"按钮进入修炼界面
4. 在修炼界面可以点击"返回地图"按钮回到地图页面
5. 导航栏按钮状态正确反映当前系统
